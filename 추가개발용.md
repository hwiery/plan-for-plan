# 추가개발 정리용 문서

## 1. 코드 작성 결과 바로 반영하기 
- 유저가 입력한 내용을 바탕으로 PRD 문서를 만들어줍니다. 그 문서를 바탕으로 우리는 서브 태스크?라고 하는 내용으로 세부적인 작업 필요 사항들이 작성된 것들을 볼 수 있을 겁니다. 
- 해당 서브태스크 세부 사항을 복사해서 바로 갖다 붙여서 쓸 수 있게 구현이 되었으나, 해당 작업의 질적 보증이 될 수 있을지는 모릅니다. 
- 그렇기 때문에 해당 문서를 만들 때 어떤 의도로 만들었는지, 프롬프트를 더 여러 번 수정해나가면서 문서의 질적인 보증이 될 수 있게 업데이트 해나가야 할 것 같습니다. 

## 2. 추후 이 자체가 우리 플랫폼 내에서 진행될 수도? 
- 궁극적으로는 바이브 코딩 웹 서비스가 되는 것입니다. 
- 유저의 요구사항을 몇 개만 받았고, 그것들을 크로스 체크하는 과정에서 프로젝트의 깊이가 나타나게 됩니다. 유저가 어떤 정보까지 알고 있는지를 알 수 있고, 유저가 원하는 플랫폼이나 기술스택으로 연동될 수 있습니다. 
- 그것들을 내부 터미널이나 프로젝트에서 돌리게 되면서 유저마다의 프로젝트를 구체적으로 빌딩해주는 서비스로 연결되게하고, claude 4 소넷 같은 llm모델들과 연동하여 내부적으로 해당 부분을 구축할 수 있게 구현한다면 많은 유저들이 높은 신뢰도를 갖고 그것을 하나하나 해결해 나갈 것입니다. 

## 3. 내가 뭘 만들지 모를 때를 위한 업데이트 
- 만약 내가 만들어야할 것이 뭔지도 모른다면 이를 추가적으로 좀 표현해주는 방식을 추가 구현합니다. 
- 유저에게 역질문할 때 기술적인 부분에 대해서 몇 개의 질문을 스킵한다면, 당신이 만들고싶은게 정확히 어떤건지 모르면 이런건 어떤가요 하면서, 가장 보편적으로 서비스를 런칭하기 좋은 방식으로 제안하는 것입니다. 
- 그것의 성공률은 어느정도가 될지 모르겠으나 결과적으로 단순하면서도 빠르게 결과를 확인하기에는 충분할 것입니다.
